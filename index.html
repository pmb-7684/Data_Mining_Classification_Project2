DataSet¶

The dataset, Default of Credit Card Clients is available from UC Irvine Machine Learning Repository. It is based on research paper written by Che-hui Lien Yeh in 2009 which examined default patterns of Taiwanese customers. The research compared the predictive accuracy of six data mining techniques. For their initial research, neutral network more accurately estimated the real probability of default.

The dataset is only available in Excel (.xls) format and contains characteristics which could help us to determine if a person is creditworthy or not. It contains 30,000 observations and 23 features. These features are ID, LIMIT_BAL, SEX, EDUCATION, MARRIAGE, AGE, PAY_0, PAY_2, PAY_3, PAY_4, PAY_5, PAY_6, BILL_AMT1, BILL_AMT2, BILL_AMT3, BILL_AMT4, BILL_AMT5, BILL_AMT6, PAY_AMT1, PAY_AMT2, PAY_AMT3, PAY_AMT4, PAY_AMT5, PAY_AMT6, and default payment next month.

The ID is the unique identifier given to each observation. LIMIT_BAL is the amount of income for the individual and their family. SEX is the gender of the individual (1 - male, 2 - female). EDUCATION is level of education (1 - graduate school, 2 - university, 3 - high school, 4 - other). MARRIAGE is martial status (1 - married, 2 - single, 3 - other)AGE is age in years. Monthly past payment history from April to September 2005 (PAY_0 - PAY_6). It was measured on a repayment scale (-1 - payment on time, 1 - one month late, 2 - two months late, 3 - three months late...,8 - eight months late, and 9 - 9 months or longer late). The billing statement balance (BILL_AMT1 - BILL_AMT6) from September April 2005 to September 2005. The previous payment amount (PAY_AMT1 - PAY_AMT6) from September April 2005 to September 2005.

Default payment next month is our target or response variable. It indicated if an individual defaulted on the next month's payment or not. (1 - Yes, 0 - No)

Model 1: Decison Trees¶

Decision Trees is a simple boolean decision rules to split each feature in the data and predict the outcome. These trees tend to work well with unbalanced datasets. This spliting (partitioning) is continued until the optiminal criteria is met. Decision tree can be used for classification and regression problems.

This model was chosen because it was the first classification model I learned. I expected and it performed poorly.

pros:

  • Easy to interpet and understand
  • Handles missing values
  • Handles non-linear
  • Handles categorical values

cons:

  • prone to overfitting
  • does not handle changes in data well
  • does not handle unbalanced class

Model 2: Random Trees¶

Random forest is a good model because of its ability to reduce the risk of overfitting.

The Random Forest is a supervised learning algorithm. It creates decision trees during the training step of the model. Each new tree is created by randomly selecting observations and features from the dataset. This random selection will introduce variability, improve prediction and reduce the risk of overfitting.

In classification, each tree "votes" based on the class (1 or 0 for this project). The forest composed of all of the tree's "votes" and majority wins. The idea is even if a few trees grow to produce errors, the entire forest will make a better decision. The classification algorithm is available in regression and classification.

pro:

  • High accuracy
  • Handles missing values and outlier
  • Less prone to overfitting
  • Works well with noise, outliers
  • Works well with unbalanced data
  • Handles numerical and categorical data

cons:

  • Can be difficult to interpret
  • Can be prone to overfitting
  • Computational complex and can take longer to make predictions

Model 3: Downsampling with Random Forest¶

We noted above that our data is unbalanced. About 78% of our observation did not default on the next month's payment, while 22% did default. Another way to address unbalanced data is to either undersample or oversample our dataset. In undersampling, we need to remove observations for the majority class until it equals the minority class. In oversample, we will create random duplicates of the minority class to equal the majority class.

Since it would be necessary to create over 10,000 observations (duplicate), run the risk of overfitting, and could possibly change the composition of the data, we undersampled to see if we get better results.

To ensure that we are pulling from the same training and testing set, we combined the training and testing datasets, created two series separating the majority and minority classes and sampled from the majority class until the count equals the minority. Both classes should have 6,636 observations, however, both ended up having 5,279 observations. Honestly, I am not sure why that was the case but I will be looking into it further.

Model 4: Naive Bayes¶

Naive Bayes uses a probabilistic machine learning algorithm. It is "naive" because it assumes that all features are independent of each other. It based on conditional probability and based on the works of Rev. Thomas Bayes. BernoulliNB classifier is suitable for discrete data and is designed for binary/boolean features.

pros:

  • Good performance with high dimensional data
  • Not sensitive to irrelevant features
  • Scalable

cons:

  • Conditional independence assumption does not always stay true.
  • Zero probability problem

Modeling 5: Logistic Regression¶

Logistic regression is regression model (not a classifier) that predicts the value of a target variable through a linear combination of features. The value is bounded between 0 and 1. It is the probability of success divided by the probability of failures known as log odds. For a binary classification, a probability less than .5 will predict 0. A probability greater than .5 will predict 1.

pros:

  • Feature scaling not needed
  • Hyperparameters tuning not needed

cons:

  • poor performance on non-linear data
  • poor performance on with irrelevant features

Model Evaluation¶

For this project, we used Decision trees, Random Forest, and Random Forest with downsizing, and Naive Bayes using Bernoulli, and logistic regression.

Based on the accuracy, basic Random Forest, Logistic regression and Naive bayes performed the best with approximately 81%.

Since the data was unbalanced, we need to look at other metrics - precision, recall, or F1 score.

Precision is the number of correctly labeled features divided by the total number of features.

Recall is the number of correctly labeled featured divided by the number of features that should have been created.

F1 score is the weighted average of precision and recall. "Accuracy in making positive predictions is measured by a recall, while identifying all positive occurrences in the data is quantified by precision. The F-score ranges from 0 to 1, with higher values indicating better performance."

The best option for imbalanced data is using the F1 score which allows us to look at the best of both worlds - precision and recall. The scores are:

  • Decision Trees 40.3%
  • Random Forest 46.3%
  • Random Forest (downsampling) 68.4%
  • Naive Bayes 49.0%
  • Logistic Regression 37.1%

The downsampling Random Forest scored the best. It was able to predict almost 69% which is better than a random guess, but it has room for improvement. Then working with the entire dataset, the results were similar with Naive Bayes performing at 49.0%. Again, this is worst then guessing.

Conclusion¶

The desired impact from this project was to successfully create a model that would predict if an individual would default on the next month's payment or not. Based on the analysis, using downsizing on random forest performed the best.

A business’s ability to predict creditworthiness can save them time and money. On the other side, these models could cause a problem for the individual consumer. The algorithm is using a small window of time to decide if a person is creditworthy. It does not take into consideration other details that could impact one's ability to pay. For example, a person has years of great repayment, and he/she becomes sick, missed payments for two months, but now their repayment schedule is back to normal. The algorithm would likely consider them a risk. Many models are based on history and it's possible that situations could change.

Based on the results, these models were not the best for this project. I plan to revisit this project and use more complex models until I can create an adequate model. In addition, I plan to learn better techniques to help me with these types of datasets. I tried to use SVM which ran for over an hour without producing a result, so I stopped it.

Honestly, 30,000 observations don’t come close to 'big data'.
